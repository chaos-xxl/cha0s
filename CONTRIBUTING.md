# Contributing to Doctor Chaos

Thanks for your interest in Doctor Chaos! This document explains how to engage with the project.

## Current State: Early Development

Doctor Chaos is in **early development**, pre-v0.1.0. During this phase:

- ❌ **Issues are currently disabled** — the core API is still being designed, and incoming issues would mostly be "when will X be ready?" which this roadmap already answers.
- ❌ **External PRs are not accepted yet** — a fast, stable API design requires single-author velocity right now.
- ✅ **Watch / Star the repo** if you want to follow along — we'll open Issues and Discussions at v0.1.0.
- ✅ **Follow [@Chaosxinglong](https://x.com/Chaosxinglong)** on X for development updates and design decisions.

## After v0.1.0

Once v0.1.0 ships, the project will open up:

- GitHub Issues will be enabled for bugs, feature requests, and design discussions.
- External PRs will be reviewed on a rolling basis.
- A dedicated `CONTRIBUTING.md` will replace this file with:
  - Dev setup instructions
  - Code style (Prettier + ESLint configured)
  - Commit convention (Conventional Commits)
  - Test requirements
  - PR review process

## In the Meantime

If you have strong thoughts on the project, DM [@Chaosxinglong](https://x.com/Chaosxinglong) on X. I read everything.
